| User Story | Feature | API / Data Source | Acceptance Criteria |
| :--- | :--- | :--- | :--- |
| As a **fan**, I want to find match schedules and navigate to stadiums easily. | Event Discovery & Real-Time Navigation | **API**: Lambda proxies for FIFA API, Google Maps API, and MARTA GTFS feed.<br>**Data**: DynamoDB for cached schedules; Google/MARTA for real-time transit. | User can view match times/locations. User can select a venue and receive real-time walking and MARTA directions. Transit ETAs are accurate within 2 minutes. |
| As a **tourist**, I want bilingual support and local recommendations near events. | Bilingual Support & Nearby Attractions | **Framework**: `next-i18next` for localization.<br>**API**: Lambda proxy for Google Places API. | The entire UI can be toggled between English and Spanish. User can view their location on a map and see a list of nearby restaurants and attractions with ratings. |
| As a **business**, I want to promote offers and fan experiences within the app. | (Future Iteration) Business Promotion Platform | **API**: (Future) A dedicated API for businesses to submit content.<br>**Data**: (Future) DynamoDB table for storing promotional content. | A business can create an account and publish geofenced offers that appear to users within a specified radius of their establishment. |
| As a **city partner**, I want real-time data to manage crowd flow and logistics. | (Future Iteration) Analytics Dashboard | **Data**: Amazon CloudWatch metrics and logs.<br>**Monitoring**: A Datadog or CloudWatch Dashboard. | A city partner can access a secure dashboard to view anonymized, real-time user location data and density heatmaps around key venues. |
